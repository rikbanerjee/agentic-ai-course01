# PM Addons — Per-Module Companions

Each technical module has a companion file here. Read the PM addon **alongside** the module (not instead of it) to get:

- **Plain-English summary** — what this module is actually about, no jargon.
- **Why it matters for the product** — the business stake.
- **What PMs decide vs what engineers decide** — clear ownership.
- **Stakeholder talking points** — language you can use in meetings.
- **Risk lenses** — what could go wrong and what to watch.
- **Decision frameworks** — when to do X vs Y, with thresholds.
- **Reading order** — what to actually read in the technical module and what to skip.

These addons are designed for both non-technical PMs (who want to understand the discipline without coding) and technical PMs (who want shortcuts to the decision points).

## Suggested PM reading paths

**Path A — "I need to scope a project this week"** (~90 min):
00 → 03 → 05 → 08 PM addons + skim [system memo](../pm-artifacts/system-memo.md) and [eval spec](../pm-artifacts/eval-and-guardrails-spec.md).

**Path B — "I'm new to AI products"** (~3 hours):
00 → 01 → 03 → 04 → 05 → 06 → 07 → 08 (all PM addons) + [glossary](../course/glossary.md).

**Path C — "I'm a Technical PM ready to spec a feature"** (~5 hours):
All 12 PM addons + read the full [RFC](../pm-artifacts/rfc-agentic-search.md), [eval spec](../pm-artifacts/eval-and-guardrails-spec.md), and [decision log](../pm-artifacts/decision-log.md).

**Path D — "I'm presenting to leadership"** (~30 min):
[Stakeholder one-pager](../pm-artifacts/stakeholder-one-pager.md) + [capstone design review](../pm-artifacts/capstone-design-review.md) + 08 PM addon.
